# 판매관리 시스템 — 전체 스펙 문서
> Claude Code 개발용. 이 문서 기반으로 전체 시스템을 구현한다.

---

## 1. 프로젝트 개요

### 목적
한국에이원 / 금화 / 라성 3개 회사의 매입·매출·마진을 한 곳에서 관리.
입고 물량과 단가를 입력하면 각사 마진·매출·계산서 발행 지시가 자동 계산된다.

### 기술 스택
- **Frontend + Backend:** Next.js (App Router)
- **DB + Auth:** Supabase (PostgreSQL + Supabase Auth)
- **배포:** Vercel
- **언어:** TypeScript

### 사용자
- **입력자:** 1명 (관리자). 모든 데이터 입력 및 수정 권한.
- **열람자:** 한국에이원 대표 / 금화 대표 / 라성 대표. 동일 계정, 동일 화면.
- **로그인:** 단일 ID/비밀번호. 로그 불필요.
- **보안:** 로그인 없이 접근 불가. Supabase Auth로 처리.

## 3. 공통 마진 배분 룰

**전 품목 공통 적용:**
- 총 마진을 한국에이원 / 금화 / 라성이 1/3씩 나눔
- 소수점 나머지(나누어 떨어지지 않는 경우)는 라성이 가져감
- 예: 100원 → 한국에이원 33원 / 금화 33원 / 라성 34원
- 한국에이원이 금화·라성에 커미션 형태로 지급
- **상무님 배분은 폐지. 시스템에 포함하지 않음**

## 4. 품목별 거래 구조

### 4-1. AL-35B / AL-65B (동국제강)

**거래 체인:** 동국제강 → 한국에이원 → 금화 → 화림

**입찰:** 기간별 낙찰. 낙찰 단가는 화림에서 제공. 6개월 단위.

**계산서 발행:**
- 동국제강 → 한국에이원: 익월 1일, 당월말 기준 **역발행**
- 금화 → 한국에이원: 익익월 1일, 익월말 기준 발행
- 화림 → 금화: **당월말 기준** 계산서 발행 (대금 수령은 익익월 1일)

**대금 흐름:**
- 동국제강이 한국에이원에 익월 말일 현금 지급
- 익익월 1일: 한국에이원 → 금화 → 화림 순차 지급

**커미션:**
- 라성: 익익월 10일, 한국에이원에서 계산서 발행 후 현금 수취

**VAT:** 10% 적용

**호진 배분 (AL35B 특이사항):**
- 화림이 물량 배분 단가 결정 (예: 100톤 중 80톤 100원, 20톤 20원)
- 호진이 물량을 더 가져갈 때: 화림 → 한국에이원 커미션 지급
  - 계산서: 당월말 기준 (익월 2~3일에 정리 후 당월말 기준으로 발행)
  - 한국에이원 → 금화·라성 배분: 익월 10일
- 호진이 물량을 덜 가져갈 때: 한국에이원 → 호진 배분 지급 (화림이 단가 결정)

**마진 계산:**
- 총 마진 = (동국 판매단가 - 화림 원가단가) × 물량
- 1/3씩 배분 (라성이 나머지)

### 4-2. 소괴탄 (동국제강)

**거래 체인:** 동국제강 → 한국에이원 → 렘코

**계산서 발행:**
- 동국제강 → 한국에이원: 익월 1일, 당월말 기준 발행
- 한국에이원 → 렘코: 동일 기준 발행

**대금 흐름:**
- 동국제강이 익월 10일 한국에이원에 현금 지급
- 한국에이원 → 렘코: 익월 10일 당일 즉시 지급

**커미션:**
- 금화·라성: 익월 15일 커미션 지급

**VAT:** 없음

### 4-3. 분탄 (동국제강)

**거래 체인:** 동국제강 → 렘코 → 한국에이원 → 동창

**계산서 발행:**
- 익월 1일, 당월말 기준으로 체인 전체 동시 발행
  (동국→렘코 / 렘코→한국에이원 / 한국에이원→동창 동시)

**대금 흐름:**
- 동국제강이 익월 10일 렘코에 지급
- 익월 10일 당일: 렘코 → 한국에이원 → 동창 순차 즉시 지급

**커미션:**
- 금화·라성: 익월 15일 커미션 지급

**VAT:** 10% 적용

### 4-4. 페로실리콘 — FeSi75 / FeSi60 (동국제강)

> FeSi75와 FeSi60은 별도 품목으로 각각 독립 관리.
> 입찰 결과에 따라 둘 다 낙찰, 둘 다 미낙찰, 하나만 낙찰 가능.

**거래 체인:** EG(중국) → 한국에이원 → 동국제강

**입찰:** 6개월 단위. EG 매입 단가 고정 (입찰 시 확정).

**환율:**
- BL(선하증권) 기재 날짜 기준 환율 적용
- 동국제강이 환율 결정. 동일 환율을 EG와의 거래에도 적용
- 환차손익 관리 불필요

**계산서 발행:**
- 동국제강 → 한국에이원: 입고 확인 후 **역발행** (BL 날짜 기준 환율 적용)
- EG → 한국에이원: 동국 역발행 계산서 발행과 동일 날짜 기준으로 발행. 달러 계산서.

**대금 흐름:**
- 동국제강 → 한국에이원: 입고 후 10일 이내 현금 지급
- 한국에이원 → EG: 입고 후 15일 이내 현금 지급 (달러 송금)

**마진 계산:**
- 총 마진 = (동국 판매금액 원화) - (EG 매입금액 원화 환산)
- 1/3씩 배분 (라성이 나머지)
- 한국에이원 → 금화·라성 커미션 지급 타이밍: EG 지급 완료 후

**VAT:** 동국 역발행 계산서 발행일 기준 동일 날짜 적용

---

### 4-5. AL-30 (현대제철)

**거래 체인:** 화림 → 한국에이원 → 현대제철

**입찰:** 6개월 단위. 낙찰 단가 = 현대제철 판매단가. 화림 매입단가는 한국에이원-화림 협의.

**계산서 발행 (마진 부분):**
- 현대제철 → 한국에이원: **10일 단위 역발행** (월 3장). 60일 어음 발행.
- 한국에이원 → 화림: 당월 3장 합산하여 **1장**으로 당월말 기준 계산서 발행
  - 예: 1회 100원 + 2회 100원 + 3회 100원 → 화림에 300원 계산서 1장

**대금 흐름 (마진 부분):**
- 어음 만기: 계산서 발행일 기준 정확히 60일
- 어음 현금화 후 익익월 1일 화림에 현금 지급
- 마진 = (현대제철 매출단가 × 물량) - (화림 매입단가 × 물량)
- 익익월 기준 계산서 발행 후 익익월 10일 금화·라성에 1/3씩 지급

**부족분 커미션:**
- 화림이 매월 부족분 물량·단가를 결정해서 한국에이원에 통보
- 화림 → 한국에이원: 당월 기준 계산서 발행 후 5일 내 현금 지급
- 마진 1/3 룰 동일 적용
- 한국에이원 → 금화·라성: 익월 기준 계산서 발행 후 익월 10일 각각 현금 지급

**화면 표시:**
- 마진(익익월 기준)과 커미션(당월 기준)을 **같은 화면에서 월별로 함께 표시**
- 비고란에 "N월 마진" / "N월 부족분 커미션" 명시

**VAT:** 10% 적용

## 5. 계산서 발행 지시 기능

### 동작 방식
1. 입고 데이터 입력 시 백그라운드에서 자동 계산 항상 유지
2. 관리자가 "발행 지시 생성" 버튼을 누를 때 해당 월 지시 목록 확정
3. 지시 목록: **누구에게 / 얼마 / 언제까지** 자동 생성

### 지시 목록 항목
- 발행 대상 회사
- 수취 회사
- 금액 (공급가액 / VAT / 합계)
- 발행 기준일
- 발행 마감일
- 지급 예정일
- **지급 완료 체크박스** (체크 시 미지급 잔액에서 제외)

### 미지급 잔액 추적
- 지급 완료 체크가 안 된 건은 미지급으로 집계
- 대시보드에서 회사별 미지급 잔액 한눈에 확인

## 6. 화면 구성 (페이지)

### 6-1. 대시보드 (메인)
- 이번 달 전체 매출 / 총 마진 요약
- 한국에이원 / 금화 / 라성 각사 이번 달 배분 마진
- 미지급 잔액 현황 (회사별)
- 이번 달 계산서 발행 지시 목록 (지급 완료 체크 포함)
- 이번 달 대금 수령·지급 일정 타임라인

### 6-2. 품목별 입고 입력
- 월 선택
- 품목 선택
- 적용 낙찰 단가 선택 (해당 기간 자동 필터)
- 물량 입력 (같은 달 복수 단가는 건별로 나눠 입력)
- 입력 시 마진 실시간 미리보기

### 6-3. 낙찰 단가 관리
- 품목별 낙찰 기간 / 판매단가 / 원가단가 등록
- 기간 겹침 방지 validation
- 페로실리콘은 달러 단가 + 환율 입력

### 6-4. 각사 매출·마진 현황
- 월별 / 품목별 필터
- 한국에이원 / 금화 / 라성 3사 나란히 표시
- 매입 / 매출 / 마진 / 커미션 수취 각각 표시
- 현대제철 마진·커미션 같은 화면에서 월별로 표시 (비고 포함)

### 6-5. 계산서 발행 지시 목록
- 월별 전체 발행 지시 목록
- 지급 완료 체크
- 미지급 잔액 집계

### 6-6. 거래처별 지급 이력
- 렘코 / 동창 / EG / 화림 각각
- 지급 이력 + 미지급 잔액
- EG는 달러·원화 병행 표시

### 6-7. 현대제철 전용 뷰
- 10일 단위 계산서 3장 전체 표시 (합산 아닌 건별)
- 월 소계 별도 표시
- 60일 어음 만기일 자동 계산 표시
- 부족분 커미션 입력 및 표시

## 7. DB 스키마 (Supabase PostgreSQL)

### products (품목)
```
id, name, display_name, buyer (납품처), unit, price_unit,
chain (jsonb), invoice_rule (jsonb), payment_rule (jsonb),
commission_rule (jsonb), vat, memo, is_active
```

### contracts (낙찰 단가 계약)
```
id, product_id, start_date, end_date,
sell_price, cost_price, currency (KRW/USD),
exchange_rate_basis, margin_distribution (jsonb), memo
```

### deliveries (납품 건)
```
id, year_month, product_id, contract_id,
quantity_kg, addl_quantity_kg, addl_margin_per_ton,
memo, created_at
```

### invoice_instructions (계산서 발행 지시)
```
id, year_month, product_id, delivery_ids (jsonb),
from_company, to_company,
supply_amount, vat_amount, total_amount,
invoice_basis_date, issue_deadline, payment_due_date,
is_paid, paid_at, memo
```

### hyundai_transactions (현대제철 전용)
```
id, year_month, invoice_date, invoice_seq (1~3),
quantity_kg, sell_price, cost_price,
supply_amount, vat_amount,
bill_due_date (발행일 +60일),
commission_amount, commission_type (margin/shortage),
margin_distribution (jsonb), memo
```

### fx_rates (환율 기록 - 페로실리콘용)
```
id, bl_date, product_id, rate_krw_per_usd, memo
```

## 8. 개발 우선순위

1. **1단계 (핵심):** 로그인 + 품목 설정 + 낙찰 단가 등록 + 입고 입력 + 마진 자동 계산
2. **2단계 (운영):** 계산서 발행 지시 생성 + 지급 완료 체크 + 미지급 잔액
3. **3단계 (분석):** 각사 매출·마진 현황 대시보드 + 거래처별 지급 이력
4. **4단계 (특수):** 현대제철 전용 뷰 + 페로실리콘 달러 처리 + 호진 배분
