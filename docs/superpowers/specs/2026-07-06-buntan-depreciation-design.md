# 분탄 월별 감가 (미배분 보관 → 렘코 반환) 설계

날짜: 2026-07-06 (같은 날 방향 정정: 렘코 차감 → **동창 차감** — 실제 동창 세금계산서 확인으로 확정.
이후 정산 상대도 정정: 동창 지급 → **렘코 반환**)
상태: 구현·배포 완료, 2026-07 실데이터(감가 180,851원) 검증 완료

## 배경

렘코가 코스닥 상장을 위해 총액 기준 매출을 유지해야 하므로, 동국제강의 분탄 감가가
렘코 구간에는 반영되지 않는다. 감가는 동창 매입 구간에서 차감하고, 그 금액은 3사
배분에서 제외해 보관했다가 계약(2026-07~09) 종료 후 렘코에 한번에 반환한다.

### 확정된 돈 흐름 (예: 계약금액 100원, 감가 10원)

물건: 동창 → 한국에이원 → 렘코 → 동국제강

| 구간 | 입금액 | 계산서 |
|---|---|---|
| 렘코 → 한국에이원 (역발행 매출) | 100원 | **100원 (총액)** |
| 한국에이원 → 동창 (매입) | 90원 | **90원 (총액 − 감가)** |

- 계산서는 입금 흐름과 동일 금액으로 발행. 동창은 차감된 금액으로 종결(최종 부담자 = 동창).
- 한국에이원이 보관한 감가(10원)는 **3사 커미션 배분에서 제외**되어 통장에 잔류 —
  계약 종료(9월) 후 **렘코에 한번에 반환**. → **최종 마진은 총액 기준으로 불변.**
- 감가는 동국이 월말에 일괄 통보 → **품목×월 단위 1건**으로 관리 (납품 건별 아님).
- 소괴탄(SOGGAE)은 변경 대상 아님 — 현행(건별 감가, 매출·매입 양쪽 차감) 유지.

## 데이터

### 테이블 `monthly_depreciations` (적용 완료)

| 컬럼 | 타입 | 설명 |
|---|---|---|
| id | uuid PK | |
| product_id | uuid FK → products | 범용 |
| year_month | text `'2026-07'` | **납품월(year_month) 기준** |
| amount | numeric | 감가 금액(원), > 0 |
| memo | text null | |
| settled_at | timestamptz null | 렘코 반환(정산) 완료 시각. null = 미정산 |
| created_at | timestamptz | |

- unique(product_id, year_month) — 월 1건 강제.
- 기존 `deliveries.depreciation_amount`는 소괴탄 전용으로 유지.

## 계산서 (`src/lib/invoice-generator/coal.ts` genBuntan)

- **매출(렘코 역발행)**: `Σ(sell_price × qty/1000) − 건별감가` — monthlyDep **미차감** (총액)
- **매입(동창)**: `Σ(cost_price × qty/1000) − 건별감가 − monthlyDep` (메모에 "월감가 X원 차감")
  - **VAT는 라인별 절사 합산**: `floor(총액×0.1) − floor(감가×0.1)` (`vatOverride`).
    차감된 공급가액에 일괄 10% 절사하면 동창 실계산서와 1원 어긋남 —
    2026-07 실계산서(공급 392,330,439 / VAT 39,233,044 / 합계 431,563,483)로 검증.
- **커미션(금화/나성)**: `splitMargin(매출총액 − 매입총액)` — **monthlyDep 제외** 기준.
  계산서 금액 차이(100 − 90)로 계산하면 감가까지 3사에 배분돼 렘코 반환 재원이
  사라짐. 감가는 미배분 상태로 통장에 잔류해야 하므로 실질 마진은 총액 기준이 맞음.
- 건별 감가(과거 월 데이터)는 기존처럼 양쪽 차감 유지 → 월별 감가 데이터가 없는
  과거 월은 재생성해도 금액 불변.

## 파이프라인·재생성

- `fetchInvoiceInputs`가 조회된 납품월들의 `monthly_depreciations`를 함께 반환 →
  `generateInvoices(deliveries, ym, monthlyDeps)` → BUNTAN 그룹에 매칭 감가 전달.
- 감가 저장/수정/삭제 서버 액션(`depreciation-actions.ts`)이 해당 지급월 계산서를
  서버사이드 재생성(계산서 없으면 스킵 — 최초 생성 시 자동 반영).

## invoices 페이지 — "분탄 감가 정산(미배분 보관 → 렘코 반환)" 패널

- 월별 감가 입력(월·금액·메모) + 목록 + **미정산 누계**(`settled_at IS NULL` 합계).
- 계약 종료 후 렘코 반환 시 "정산완료" → `settled_at` 기록, 취소 가능.
- 렘코 반환 시점의 계산서 발행 방식은 미확정 — 확정되면 추가 (범위 제외).

## analytics

- 분탄 행: **매입 = 총액 − 감가**, **매출 = 총액**, **마진·3사 배분 = 총액 기준 불변**.
- 매입 셀에 "감가 −X" 배지 (`ProductRow.depreciationKrw`).

## 납품 폼

- BUNTAN은 건별 감가 입력란 숨김(월별 안내 문구). 과거 건별 감가 보유 건 수정 시에만
  노출(유실 방지). SOGGAE 현행 유지.
