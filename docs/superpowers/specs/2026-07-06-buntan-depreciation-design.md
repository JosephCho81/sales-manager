# 분탄 월별 감가 (렘코 미수) 설계

날짜: 2026-07-06
상태: 사용자 승인 대기

## 배경

렘코가 코스닥 상장을 위해 총액 기준 매출을 유지해야 하므로, 동국제강의 분탄 감가를
계약 기간(2026-07~09) 중에는 하류로 전가하지 않고 한국에이원이 선부담한다.

### 확정된 돈 흐름 (예: 계약금액 100원, 감가 10원)

물건: 동창 → 한국에이원 → 렘코 → 동국제강

| 구간 | 입금액 | 계산서 |
|---|---|---|
| 동국 → 렘코 | 90원 | (시스템 외부) |
| 렘코 → 한국에이원 (역발행 매출) | 90원 | **90원 (총액 − 감가)** |
| 한국에이원 → 동창 (매입) | 100원 | **100원 (총액)** |

- 계산서는 입금 흐름과 동일 금액으로 발행.
- 한국에이원이 선부담한 감가(10원)는 **렘코 미수금**으로 누적, 계약 종료(9월) 후
  렘코가 한번에 지급. → **최종 마진은 총액 기준으로 불변.**
- 감가는 동국이 월말에 일괄 통보 → **품목×월 단위 1건**으로 관리 (납품 건별 아님).
- 소괴탄(SOGGAE)은 이번 변경 대상 아님 — 현행(건별 감가, 매출·매입 양쪽 차감) 유지.

## 데이터

### 신규 테이블 `monthly_depreciations`

| 컬럼 | 타입 | 설명 |
|---|---|---|
| id | uuid PK | |
| product_id | uuid FK → products | 분탄 외 품목도 쓸 수 있게 범용 |
| year_month | text `'2026-07'` | **납품월(year_month) 기준** |
| amount | numeric | 감가 금액(원), > 0 |
| memo | text null | |
| settled_at | timestamptz null | 렘코 정산(회수) 완료 시각. null = 미정산 |
| created_at | timestamptz | |

- unique(product_id, year_month) — 월 1건 강제.
- 기존 `deliveries.depreciation_amount`는 소괴탄 전용으로 유지. 분탄 신규 입력에서는
  사용하지 않음(과거 데이터는 그대로 보존).

## 계산서 (`src/lib/invoice-generator/`)

### `coal.ts` genBuntan 변경

시그니처에 해당 납품월의 월별 감가 금액(`monthlyDep: number`, 없으면 0)을 추가.

- **매출(렘코 역발행)**: `Σ(sell_price × qty/1000) − 건별감가 − monthlyDep`
- **매입(동창)**: `Σ(cost_price × qty/1000) − 건별감가` — monthlyDep **미차감** (총액 지급)
- **커미션(금화/나성)**: `splitMargin(매출총액 − 매입총액)` — **monthlyDep 제외** 기준.
  계산서 금액 차이(90 − 100)로 계산하면 커미션 과소지급. 감가는 계약 종료 후 전액
  회수되므로 실질 마진은 총액 기준이 맞음.
- 건별 감가(과거 월 데이터)는 기존처럼 양쪽 차감 유지 → 월별 감가 데이터가 없는
  과거 월은 재생성해도 금액 불변 (지급완료 이력 정합, 날짜 분기 불필요).
- genSoggae 무변경.

### 파이프라인 (`invoice-data.ts`, `index.ts`)

- `fetchInvoiceInputs(yearMonth)`: `monthly_depreciations`를 함께 조회해
  `InvoiceInputs`에 추가. 분탄은 offset=1이므로 지급월 M의 계산서는 납품월 M−1 —
  감가 조회도 납품월 기준으로 매칭.
- `generateInvoices` 라우팅에서 BUNTAN 그룹에 해당 납품월 감가를 전달.

### 재생성 정합

- 감가 저장/수정/삭제 서버 액션이 성공하면 해당 지급월(납품월+1) 계산서를
  서버사이드 재생성(`replaceInvoices` 기존 플로우 재사용). 계산서가 아직 없으면
  스킵 — 이후 최초 생성 시 fetchInvoiceInputs가 감가를 포함하므로 자동 반영.
- `regen-check.ts`는 확장하지 않음 (액션 트리거 방식으로 충분).

## invoices 페이지 — "분탄 감가 정산(렘코 미수)" 패널

입력과 조회를 한곳에서:

```
2026-07   10원   메모   [정산전]
2026-08    5원          [정산전]
──────────────────────────────
미정산 누계   15원
```

- 월별 감가 입력(품목=분탄 고정 노출, 월·금액·메모), 수정·삭제.
- 미정산 누계 = `settled_at IS NULL` 합계.
- 계약 종료 후 렘코 입금 시 "정산완료" 버튼 → `settled_at` 기록, 누계에서 제외,
  행은 이력으로 남음. 실수 대비 "정산취소"(settled_at → null)도 제공.
- 렘코 정산 시점의 계산서 발행 방식은 미확정 — 확정되면 별도 추가 (이번 범위 제외).

## analytics

- `buildAllAnalytics`에 월별 감가를 전달.
- 분탄 행: **매출 = 총액 − 감가** (계산서·실입금 기준), **매입 = 총액**,
  **마진·3사 배분 = 총액 기준 유지** (회수 예정이므로 불변).
- 분탄 행에 해당 월 감가를 "미수 감가 −X원" 배지로 표시 — 매출과 계산서가
  총액과 다른 이유를 화면에서 알 수 있게.
- 소괴탄 등 건별 감가 집계(양쪽 차감)는 현행 유지.

## 납품 폼 (`deliveries/`)

- BUNTAN 선택 시 건별 감가 입력란 숨김 + "감가는 지급 스케줄의 월별 감가에서 입력"
  안내 문구. SOGGAE는 현행 유지. (이중 입력 방지)
